❯ why python3 app.py not working but venv/bin/python3 working?                  

⏺ Your Mac has multiple Python installations:

  ┌──────────────────────┬─────────────────────────────────────────┬───────┐
  │       Command        │                Points to                │ Versi │
  │                      │                                         │  on   │
  ├──────────────────────┼─────────────────────────────────────────┼───────┤
  │ python3              │ /Library/Frameworks/Python.framework/Ve │ Pytho │
  │                      │ rsions/3.7/                             │ n 3.7 │
  ├──────────────────────┼─────────────────────────────────────────┼───────┤
  │ /opt/anaconda3/bin/p │                                         │ Pytho │
  │ ython3               │ Anaconda                                │ n     │
  │                      │                                         │ 3.12  │
  ├──────────────────────┼─────────────────────────────────────────┼───────┤
  │                      │                                         │ Pytho │
  │ venv/bin/python3     │ Your venv (built from Anaconda)         │ n     │
  │                      │                                         │ 3.12  │
  └──────────────────────┴─────────────────────────────────────────┴───────┘

  When you run python3 app.py, it uses Python 3.7 — which has no Flask 
  installed.

  When you run venv/bin/python3 app.py, it uses the venv's Python 3.12 — which
  has Flask installed.

  The fix (permanent): Add Anaconda to your PATH so python3 always points to
  3.12:

  echo 'export PATH="/opt/anaconda3/bin:$PATH"' >> ~/.zshrc
  source ~/.zshrc

  After this, python3 app.py will work directly.
